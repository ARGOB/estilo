    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-estándares-web" class="anchor" href="#estándares-web" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Estándares web</h1>

<h2><a id="user-content-objetivo" class="anchor" href="#objetivo" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Objetivo:</h2>

<p>El presente documento tiene como objetivo definir las pautas elementales a ser consideradas al momento de diseñar, desarrollar e implementar todo sitio web del Estado Nacional, ya sea realizado por empleados del Estado o personal tercerizado, con el fin de ser utilizados por ciudadanos.</p>

<p>Con los siguientes estándares, se busca homogeneizar la experiencia entre los diversos activos del estado, para facilitar la comprensión y utilización, primando la Usabilidad, la Accesibilidad y la Experiencia.</p>

<h2><a id="user-content-requisitos" class="anchor" href="#requisitos" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Requisitos</h2>

<h3><a id="user-content-1-desarrollo" class="anchor" href="#1-desarrollo" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a><strong>1</strong>. <strong>Desarrollo</strong></h3>

<ul>
<li><p>Se pueden usar redirects desde dominios a otros dominios.</p></li>
<li><p>Los sitios web que sean la base del dominio deben usarse con <a href="http://www">www</a>.
Ejemplo <a href="http://www.educacion.gob.ar">www.educacion.gob.ar</a>. Si uno ingresa a <a href="http://educacion.gob.ar">educacion.gob.ar</a> se deberá generar redirect 301 al dominio con las <a href="http://www">www</a>.</p></li>
<li><p>Todos los sitios web del Estado usarán la terminación .gob.ar. El registro de .gov.ar deberá generar un redirect 301 al dominio gob.ar.</p></li>
<li><p>Todos los sitios web que ofrezcan servicios de consulta online (mostrar ejemplos) deben tener una API disponible y/o planificada su implementación. (Ver estándares de APIs).</p></li>
<li><p>Todos los sitios con sistemas de registración deben seguir los lineamientos definidos por la coordinación de identidad digital y tarjeta inteligente.</p></li>
<li><p>Los sitios deben ser responsive para poder adaptarse y poder ser navegables desde diversos dispositivos, como por ejemplo dispositivos móviles.</p></li>
<li><p>Todos los sitios y aplicaciones deberán contar con el mínimo de documentación necesaria para ser instalados en diferentes entornos.</p></li>
<li><p>Compatibilidad con los siguientes navegadores</p>

<ul>
<li><p>Internet Explorer 8 (NOTE:  Para versiones menores a IE8, debe aparecer una alerta avisando que el navegador está desactualizado, como esto: <a href="https://browser-update.org/">https://browser-update.org/</a>)</p></li>
<li><p>Chrome 19</p></li>
<li><p>Firefox 3</p></li>
<li><p>Safari 6</p></li>
</ul></li>
<li><p>Control de versiones: deberán estar etiquetadas con correspondiente nombre con el formato "vx.x.x". </p></li>
</ul>

<h3><a id="user-content-2-diseño" class="anchor" href="#2-diseño" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a><strong>2.</strong> <strong>Diseño</strong></h3>

<ul>
<li><p><strong>Aplicación de marca sobre logotipo del área</strong>: los logotipos de las áreas de gobierno deben respetar los lineamientos del Manual de identidad visual para web y aplicaciones móviles [link al manual]. Dependiendo de la cantidad de caracteres que tenga el nombre del área puede representarse con 1, 2 o 3 líneas de texto, junto a la leyenda "Presidencia de la Nación" y el Escudo de la Nación.</p></li>
<li><p><strong>Uso de encabezados</strong>: los encabezados deben tener un alto mínimo de 72px y fondo de color pleno blanco o color primario de la paleta. El logotipo debe tener un alto de 50px y ancho proporcional. Éste debe estar a 10px del margen superior y preferentemente alineado al margen izquierdo del contenedor principal.</p></li>
<li><p><strong>Uso de pie de página oficial</strong>: al final de las páginas debe agregarse el pie de página que distingue a los sitios webs oficiales del gobierno. Este debe tener la leyenda: "Dominio.gob.ar es un sitio web oficial del Gobierno Argentino". Junto a este debe estar el logotipo de Argentina.gob.ar, el cual debe linkear la página de inicio de esa web.</p></li>
</ul>

<ul>

<p>Dentro de los recursos adjuntos a este documento se pueden encontrar ejemplos de código para el encabezado y los tipos de pie de página, el favicon y las diferentes disposiciones del logotipo.</p></li>
<li><p><strong>Justificación:</strong> Brindar un sistema visual homogéneo acorde al manual de identidad y marca, entre todas las propiedades digitales del gobierno.</p></li>
<li><p><strong>Recursos:</strong> Están disponibles los recursos gráficos a través de la base para webs que busca homogeneizar y facilitar la creación de sitios web. </p></li>
</ul>

<h4><a id="user-content-colores" class="anchor" href="#colores" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a><strong>Colores:</strong></h4>

<p>La paleta de colores a utilizar es la definida en el manual de identidad visual para web y aplicaciones móviles. Los usos son los siguientes:</p>

<ul>

</ul></li>

<h4><a id="user-content-tipografías" class="anchor" href="#tipografías" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a><strong>Tipografías:</strong></h4>

<h4><a id="user-content-estética-y-elementos-gráficos" class="anchor" href="#estética-y-elementos-gráficos" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a><strong>Estética y elementos gráficos:</strong></h4>

<p>Como estética se usa un sistema visual minimalista/flat donde predomina el diseño del contenido evitando el uso de ornamentos que distraigan o afecten la performance del sitio.</p>

<p>Se recomienda el uso de fotografías como parte del contenido, ya que hace más amena la lectura y ayuda a la comunicación. Ver los estándares de contenido para más información.</p>

<li><p><strong>CSS3</strong>: Uso de las últimas versiones, siempre teniendo en cuenta el soporte de los navegadores de las diferentes propiedades.</p></li>

<h3><a id="user-content-3-usabilidad" class="anchor" href="#3-usabilidad" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a><strong>3.</strong> <strong>Usabilidad</strong></h3>

<ol>
<li><p>Diseñar para todas las resoluciones de pantalla de computadoras, tablets y celulares posibles.</p></li>
<li><p>Usar convenciones de estética y funcionamiento de cada plataforma o sistema operativo (Ej: <a href="https://www.google.com/design/spec/material-design/introduction.html">Google Material Design,</a> <a href="https://developer.apple.com/library/ios/documentation/UserExperience/Conceptual/MobileHIG/">iOS Guidelines.</a> Para sitios web se deben respetar sólo los lineamientos de identidad visual).</p></li>
<li><p>Entender qué funcionalidades son las más usadas (en base a medición de analíticas y pruebas de usabilidad) para hacerlas más fácil de acceder en futuras versiones.</p></li>
<li><p>Mostrar al usuario el estado del sistema en todo momento. Dónde está (títulos y breadcrumbs), lo que está haciendo y de qué se trata (títulos y descripciones), cuál es su progreso (indicadores de carga y barras de progreso, etc).</p></li>
<li><p>Brindar una respuesta inmediata a cada interacción (un cambio visual, un mensaje de carga, etc).</p></li>
<li><p>Los elementos interactivos deben tener un estilo para cuando está:</p>

<ol>
<li><p>En estado normal</p></li>
<li><p>En foco</p></li>
<li><p>Over (mouse encima del elemento)</p></li>
<li><p>Activo (cuando se está haciendo clic o tap)</p></li>
<li><p>Visitado (para enlaces dentro de un párrafo).</p></li>
</ol></li>
<li><p>Los botones o áreas interactivas deben ser grandes como para no necesitar precisión al hacer clic o tap. Esta necesidad está basada en la <a href="https://es.wikipedia.org/wiki/Ley_de_Fitts">ley de Fitts.</a> En aplicaciones móviles el mínimo de un área interactiva debe ser 44x44pts.</p></li>
<li><p>Los campos de formularios deben mostrar su etiqueta al estar completos. <a href="https://www.nngroup.com/articles/form-design-placeholders/">Ver ejemplos de uso correctos e incorrectos.</a></p></li>
</ol>

<h3><a id="user-content-4-accesibilidad" class="anchor" href="#4-accesibilidad" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>4. <strong>Accesibilidad</strong></h3>

<ol>
<li><p>Los textos deben cumplir las 5 reglas de legibilidad:</p>

<ol>
<li><p>El contraste mínimo del texto debe ser 4,5:1 (Para un fondo blanco #fff, el texto más claro posibles es #767676). Probarlo usando herramientas como por ejemplo <a href="http://webaim.org/resources/contrastchecker/">http://webaim.org/resources/contrastchecker/</a></p></li>
<li><p>El cuerpo de texto mínimo para sitios web es de 14px/pts para párrafos y 12px/pts para anotaciones.</p></li>
<li><p>Usar interlineado de párrafos de 1,5 a 2 veces el cuerpo de texto.</p></li>
<li><p>La separación entre párrafos debe ser 1,5 veces el interlineado.</p></li>
<li><p>No usar párrafos con texto justificado.</p></li>
</ol></li>
<li><p>Usar subrayado para los enlaces (a menos que sea un menú o botón).</p></li>
<li><p>Las fotos deben tener un texto alternativo descriptivo.</p></li>
<li><p>No usar texto que en vez de ser texto es una imagen.</p></li>
<li><p>Incluir una vista de impresión, ya sea usando una hoja de estilos o una página aparte, para que todo contenido se vea bien al imprimir.</p></li>
<li><p>Usar subtítulos para videos.</p></li>
<li><p>En web, permitir el uso de un teclado sin necesidad de mouse.</p></li>
</ol>

<h3><a id="user-content-5-seguridad" class="anchor" href="#5-seguridad" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>5. <strong>Seguridad</strong></h3>

<p>La comunicación con los web services debe ser encriptada usando un certificado SSL/TLS</p>

<p>Todas las URL que lanza una aplicación deben ser https.</p>

<p>Sólo se deben pedir al usuario los permisos mínimos y estrictamente los que son necesarios.</p>

<p>Los datos de los usuarios usados en la registración deben ser guardados con seguridad.</p>

<p>Limitar cantidad de intentos de logins.</p>

<p>Se deben actualizar los frameworks de desarrollo para evitar vulnerabilidades de seguridad. </p>
